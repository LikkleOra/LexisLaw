  <action>
    Create database schema with the following tables:
    
    1. **clients** - Client information
       - id (UUID, PK)
       - name (VARCHAR)
       - phone (VARCHAR) 
       - email (VARCHAR)
       - created_at (TIMESTAMP)
    
    2. **bookings** - Consultation bookings
       - id (UUID, PK)
       - client_id (FK to clients)
       - matter_type (VARCHAR)
       - preferred_date (DATE)
       - preferred_time (VARCHAR)
       - description (TEXT)
       - status (ENUM: pending, confirmed, completed, cancelled)
       - created_at (TIMESTAMP)
    
    3. **matters** - Legal matters linked to bookings
       - id (UUID, PK)
       - booking_id (FK to bookings)
       - reference (VARCHAR, unique, format: REF-XXXXX)
       - attorney_id (FK, nullable)
       - status (ENUM: pending, in_progress, awaiting_docs, hearing, resolved)
       - next_action (TEXT)
       - updated_at (TIMESTAMP)
    
    4. **attorneys** - Firm attorneys
       - id (UUID, PK)
       - name (VARCHAR)
       - specializations (VARCHAR[])
       - email (VARCHAR)
    
    Create seed data for attorneys and sample matters.
